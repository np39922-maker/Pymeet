======================================================
7. DEEP DIVE: API HIT AUR SERVER CONNECTION KAISE HOTA HAI?
======================================================

Jab aap kisi interviewer ko ye samjhayenge, toh unhe lagega ki aapko architecture ki puri knowledge hai. Isko dhyan se padhiye!

A. Frontend Backend se Baat Kaise Karta Hai? (The Big Picture)
Frontend (React) ke paas apna dimag nahi hota data save karne ke liye. Usse data Backend (FastAPI Python Server) se mangna padta hai. Is baat-cheet ko "API Hitting" kehte hain. 
Frontend ek 'Request' bhejta hai (Jaise "Ye lo email aur password, mujhe andar aane do"), aur Backend ek 'Response' deta hai ("Ye lo tumhara Token, aaja andar").

B. "Axios" Ka Use (Postman In Code)
Humne frontend me APIs ko hit karne ke liye `axios` naam ki library ka use kiya hai. Normal `fetch()` command se bhi ye ho sakta tha, par Axios me hume ek "Interceptor" ka fayda milta hai.
- Interceptor kya hai?: Ye ek guard (chowkidar) ki tarah kaam karta hai. Har baar jab bhi frontend se koi request nikalti hai, Interceptor usey raste me rokata hai, us request ke andar chupke se JWT Token (Aapka ID Card) dalta hai, aur fir use server ke paas bhejta hai. 
Iska fayda ye hai ki hume har page ke code me bar-bar token add karne ka code nahi likhna padta.

C. Step-by-Step API Hit Process (Example: Login)
Maan lijiye aapne Login button dabaya, toh piche kya hota hai?
1. React component se `axios.post('/api/auth/login', {username, password})` function call hota hai.
2. Axios is data ko JSON (JavaScript Object Notation) format me pack karke network ke raaste Python backend par bhej deta hai.
3. FastAPI (Backend) me `@app.post("/api/auth/login")` naam ka ek Route baitha hota hai, jo is request ko catch karta hai.
4. Python database (SQLAlchemy) me check karta hai ki kya ye user exist karta hai? Agar haan, toh kya password sahi hai (bcrypt decode karke check karta hai).
5. Agar sab theek hai, toh Python ek naya JWT Token banata hai aur JSON response me wapas Frontend ko bhej deta hai: `{"access_token": "ey123..."}`.
6. Frontend is token ko LocalStorage me save kar leta hai aur aapko Dashboard par le jata hai.

D. Local vs Production (Vercel) Connection
Aapke man me sawal aayega: "Mera React toh Localhost:5173 par chal raha hai aur Python Localhost:8000 par, toh dono connect kaise hote hain?"
- Local Development me: Humne Vite me "Proxy" set ki hoti hai. Jab React `/api/...` ko call karta hai, toh Vite secretly us request ko `http://localhost:8000` par bhej deta hai. Isse hume CORS (Cross-Origin) error nahi aata.
- Vercel (Production) me: Jab hum Vercel par host karte hain, toh hum backend ka asli URL (jaise Render ya kisi VPS ka server URL) `.env` file me dal dete hain. Frontend direct us live API par hit karta hai.

E. CORS (Cross-Origin Resource Sharing) Kya Hai?
Browser bahut strict hota hai. Agar aapki site (frontend.com) kisi dusri site (backend.com) se data mangti hai, toh browser use block kar deta hai (Isko CORS Error kehte hain).
Isliye hamne Python (FastAPI) me "CORSMiddleware" lagaya hai. Ye browser ko kehta hai: "Bhaiya, ye frontend hamara hi hai, isko data lene do, roko mat."

F. Socket.IO Connection (Live Connection)
API hit ek baar me khatam ho jati hai (Request gayi, Response aaya, connection close). Lekin Video Calling me hume continuous connection chahiye. 
Isliye hum API ke alawa ek aur connection banate hain jise "WebSocket" (Socket.io) kehte hain. 
Jab aap Dashboard open karte hain, toh `socket.io-client` ek pipe (tunnel) bana leta hai Python server ke sath. Ab agar koi dusra user call join karega, toh Python us pipe se direct message bhej dega (Bina API hit kiye!). 

SUMMARY FOR INTERVIEWER:
"We used Axios interceptors to automatically attach JWT tokens to every outgoing API request. Our FastAPI backend handles CORS securely and responds with JSON data. For real-time updates and WebRTC signaling, we bypass standard HTTP REST APIs and maintain a persistent duplex connection using Socket.IO."
